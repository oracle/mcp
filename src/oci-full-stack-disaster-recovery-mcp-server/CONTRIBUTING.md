# Contributing to this repository

We welcome your contributions. Before submitting code via a pull request, make
sure there is an issue tracking the fix or enhancement and that any related
documentation has been updated.

## Oracle Contributor Agreement

Before submitting code, sign the
[Oracle Contributor Agreement](https://oca.opensource.oracle.com/). Commits
must include a sign-off line using the same name and email address used for the
OCA:

```text
Signed-off-by: Your Name <you@example.org>
```

Git can add this automatically:

```bash
git commit --signoff
```

Pull requests from contributors that cannot be verified as having signed the
OCA cannot be accepted.

## Pull request process

1. Create or reference an issue that explains the bug or enhancement.
2. Fork the repository and create a branch for your change.
3. Keep the MCP server layout aligned with Oracle MCP best practices.
4. Update documentation and samples when behavior or configuration changes.
5. Run the test suite before opening the pull request.
6. Explain what changed and include simple validation steps in the pull request.

## Security

Do not open a public issue for suspected security vulnerabilities. Follow the
repository's security policy or contact the maintainers privately.
